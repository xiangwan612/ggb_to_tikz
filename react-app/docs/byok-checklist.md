# BYOK 上线前 5 分钟自查清单

> 目标：纯前端静态部署时，避免平台密钥泄露。  
> BYOK = 用户自带 API Key（Bring Your Own Key）。

## 1. 密钥来源

- [ ] 代码仓库里没有任何真实 API Key（包含 `src/`、`public/`、`README`、示例截图）。
- [ ] 没有把你的密钥写成默认值（首次打开页面时应为空）。

## 2. 前端行为

- [ ] API Key 只由用户在设置面板输入，不自动回填平台密钥。
- [ ] Key 只保存在用户浏览器本地（`localStorage` / `sessionStorage`）。
- [ ] 页面文案明确提示“密钥仅保存在本地浏览器，不会上传到你的服务器”。

## 3. 网络请求安全

- [ ] 浏览器请求头中 `Authorization` 来自用户输入，不包含硬编码密钥。
- [ ] 控制台日志、报错信息里不输出完整 Key（必要时打码显示）。
- [ ] 不把 Key 拼到 URL query 参数中（只放 Header）。

## 4. 用户可控性

- [ ] 提供“连接测试”按钮，便于用户确认 `Base URL + Key + Model` 是否可用。
- [ ] 提供“一键清除本地密钥”功能，清空后立即失效。

## 5. 部署与发布

- [ ] 静态部署平台（Vercel/Netlify/Cloudflare Pages）中未配置会暴露给前端的真实平台密钥。
- [ ] 发布说明中声明：本工具采用 BYOK，用户对自己的 Key 使用与额度负责。

## 发布前快速回归（建议）

1. 打开无痕窗口访问线上地址。  
2. 确认默认没有 Key。  
3. 输入测试 Key 调用一次，功能正常。  
4. 点击“清除本地密钥”，刷新后确认 Key 消失。  
5. 打开开发者工具检查：请求头正常，日志无明文密钥。
